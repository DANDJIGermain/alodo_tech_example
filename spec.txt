Objectif & Philosophie du Site
Le site d'ALODO TECH n'est pas un simple site vitrine. C'est une machine de crédibilité, une porte d'entrée
pour les investisseurs et partenaires, et la preuve publique que l'équipe est sérieuse, structurée et
ambitieuse.
La règle des 5 secondes
Tout visiteur doit obtenir une réponse claire à ces 3 questions en moins de 5 secondes :
01 Qui êtes-vous ? ALODO TECH — startup tech africaine
02 Que construisez-vous ? Infrastructure financière & digitale pour marchés émergents
03 Pourquoi ça compte ? Digitaliser l'économie informelle, inclure les MPME
Positionnement de marque
Le site doit transmettre l'ADN d'ALODO TECH : minimaliste, ambitieux, pan-africain. Les références
visuelles sont Apple Inc. et OpenAI.com — pas de surcharge, pas de kitsch, pas de design 'africain
classique'. Chaque pixel doit inspirer confiance.
Trois audiences principales
Audience Besoin
Investisseurs & VCs Cherchent traction, vision, équipe, opportunité de marché
Partenaires B2B Banques, IMF, gouvernements — cherchent fiabilité & solutions
Talents Développeurs, designers, business — cherchent une aventure

• Boîtier intelligent (hardware IoT) installé au point de vente

• WhatsApp Bot : interface accessible sans smartphone haut de gamme
• Score de crédit MPME basé sur les données collectées

• Captures d'écran du WhatsApp Bot
• Schéma du flux : vente → enregistrement → score → crédit

• Accès au crédit formel pour la première fois
• Visibilité financière numérique
• Croissance du chiffre d'affaires mesurable

Stack choisie pour allier performance, scalabilité, rapidité de développement et crédibilité aux yeux des
investisseurs tech.
Couche Technologie Justification
Frontend Framework Next.js 14 (App Router) SSR/SSG, SEO natif, performances, ecosystem React
UI & Styling Tailwind CSS + Framer Motion Minimalisme, animations propres, pas de CSS custom spaghetti
Composants shadcn/ui Composants accessibles, customisables, design-system ready
CMS Sanity.io (recommandé) ou Strapi CMS headless moderne, preview en temps réel, gratuit en starter
Backend / API Node.js + Express ou Next.js API RoutesLégèreté, ecosystème JS unifié, déploiement facile
Base de données PostgreSQL (Supabase) Relationnel robuste, Supabase = BaaS pratique
Auth NextAuth.js Auth simplifiée, multi-providers, sessions sécurisées
Emails Resend + React Email Emails transactionnels modernes, templates React
Analytics Google Analytics 4 + Hotjar Gratuit, standard industrie, heatmaps Hotjar
Hosting Vercel Déploiement Git push, CDN global, preview branches

Formulaires React Hook Form + Zod Validation typesafe, UX formulaires optimale
Internationalisation next-intl i18n First-class dans Next.js App Router

Design System
Le design est ce qui sépare un site de startup sérieuse d'un site amateur. Référence absolue : Apple.com et
OpenAI.com. Minimalisme, espace, typographie.
Palette de Couleurs
Rôle Couleur Usage
Primary Black #0A0A0A Textes principaux, headers, fond dark sections
Primary White #FFFFFF Fonds, cartes, espace négatif
Brand Blue #0057FF CTAs, liens, accents, icônes actives
Teal Accent #00C2A8 Impact, success states, data viz
Light Grey #F5F5F7 Fonds de sections alternées (style Apple)
Medium Grey #6E6E73 Textes secondaires, metadata, captions
Divider Grey #D1D1D6 Bordures, séparateurs
Typographie
• Display / Hero : Inter 700 ou Geist Sans Bold — tailles 48px–72px
• Titres H2/H3 : Inter 600 — 28px / 22px
• Body text : Inter 400 — 16px, line-height 1.6
• Captions / Labels : Inter 400 — 12px–14px, color #6E6E73
• Code / Mono (si besoin) : JetBrains Mono
• Fonte de fallback : system-ui pour performance
Règles de Design
n À FAIRE
• Espacement généreux (padding 80px+ entre sections)
• Animations légères au scroll (Framer Motion)
• Images haute résolution ou illustrations SVG
• Boutons avec états hover / focus clairs
• Icônes cohérentes (Lucide React)
• Grid 12 colonnes, max-width 1280px
• Sections alternées blanc / gris clair
n À ÉVITER
• Gradients multicolores criards
• Animations excessives qui distraient
• Trop de polices différentes (2 max)
• Textes longs sans respiration
• Images de stock génériques
• Fonds noirs partout (sauf hero)
• Boutons sans contraste suffisant

Erreurs Critiques à Éviter
Ces erreurs transforment un site pro en site amateur. Chaque membre de l'équipe doit les connaître.
n Tout sur une seule page Un one-pager interminable est illisible sur mobile et tue le SEO. Structure
multi-pages obligatoire.
n Design cheap ou surchargé Gradients agressifs, trop de couleurs, icônes dépareillées. Référez-vous
systématiquement à Apple.com et OpenAI.com.
n Murs de texte Aucun paragraphe de plus de 3 lignes. Si c'est plus long, c'est un article
de blog — pas une homepage.
n Vision floue Le visiteur doit comprendre ce que vous faites en 5 secondes. Si vous
devez expliquer 5 minutes, reécrivez.
n Pas de CTA clairs Chaque section doit amener quelque part. Boutons visibles, labels
d'action ('Découvrir', 'Contacter', 'Investir').
n Coder sans maquette Ne jamais ouvrir VS Code avant d'avoir des maquettes Figma validées.
Le design en browser = perte de temps garantie.
n Ignorer le mobile 60-70% du trafic africain est mobile. Mobile-first est non-négociable, pas
optionnel.
n Faux chiffres sans
disclaimer
Chiffres simulés pour prototype = OK avec mention 'Projections'. Chiffres
inventés présentés comme réels = problème de crédibilité.
n Emails non-pro contact@gmail.com détruit la crédibilité instantanément.
contact@alodo.tech depuis le jour 1.
n Pas de HTTPS / domaine
propre
Un site en http:// ou sur un sous-domaine Vercel par défaut en production
est inacceptable pour une startup sérieuse.

Roadmap de Livraison
Processus en 3 phases. Ne pas sauter les étapes. La qualité du site dépend de la rigueur du processus.
Phase 1 — Fondations Semaines 1–2
3 Définir le contenu exact de chaque page (copywriting)
3 Choisir le domaine + configurer les emails pro
3 Créer les maquettes Figma (desktop + mobile) pour toutes les pages
3 Faire valider les maquettes par l'équipe entière
3 Initialiser le repo GitHub alodo-tech-website
3 Setup Next.js 14 + Tailwind + shadcn/ui + next-intl
Phase 2 — Développement V1 Semaines 3–6
3 Intégration Figma → code (composants réutilisables d'abord)
3 Pages : Home, About, Products (ALODO + COINAI), Contact
3 Multi-langue FR/EN
3 Responsive mobile first — tests sur vrai device
3 SEO : meta tags, OG, sitemap, robots
3 Formulaire contact + leads investisseurs
3 Déploiement preview sur Vercel
Phase 3 — Enrichissement Semaines 7–10
3 Pages : Solutions, Technology, Impact, Blog, Careers, Partners
3 CMS Sanity.io branché sur Blog
3 Analytics GA4 + Hotjar
3 Démo interactive ALODO (simulateur basique)
3 Performance audit : Lighthouse > 90 sur toutes les pages
3 Lancement domaine production + annonce LinkedIn
